Plan 5 — Cake design + print pipeline v2 · image-only

Three mockups for the new surfaces. Click into each to review; come back here to navigate.

What changed from v1

  • No message text field anywhere. Both the box card and the cake print are pure images uploaded by the customer.
  • Customer uploads a raw image → drags a locked-aspect-ratio crop rectangle (4:6 for card, 1:1 for cake) → confirms. The cropper opens as a modal.
  • Per-recipient personalization happens by uploading a different image for that recipient. No automatic text overlay at print time.
  • Data model on each design = { cake_image_url, card_image_url }. No message, no signoff.
  • Print artifacts simplify to "tile the card image 8 per letter" + "zip the cake images".
